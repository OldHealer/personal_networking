### Инструкция: подключение Keycloak (локально)

Подготовьте и разверните контейнер из официального образа:

```bash
docker run -d --name rockfile-keycloak --network rockfile-network -p 8081:8080 -e KEYCLOAK_ADMIN=admin -e KEYCLOAK_ADMIN_PASSWORD=admin quay.io/keycloak/keycloak:latest start-dev
```

> ⚠️ Не рекомендуется подключать БД Keycloak к БД проекта, так как он создаст почти 100 таблиц

Перейдите на `http://localhost:8081`  
**🔐 Для входа:** `admin / admin`

Далее настройте:

- **Realm**: `rockfile`
- **Client**: `rockfile-api` (тип: `confidential`, включить `Standard Flow`)
- **Valid Redirect URIs**: `http://localhost:3000/`* (или URL фронта)
- **Web Origins**: `http://localhost:3000`
- **Roles**: `owner`, `admin`, `user`
- **Users**: тестовый пользователь для проверки авторизации

### Рекомендуемые поля в токене

- `tenant_id` — ID арендатора
- `roles` — роли пользователя
- `aud` — `rockfile-api`
- `iss` — URL realm

### Audience Mapper (чтобы `aud` был `rockfile-api`)

**Зачем:** без аудитории `rockfile-api` проверка токена в API будет падать.

1. `Client scopes` → выберите `rockfile-api-dedicated`
2. `Mappers` → `Create mapper`
3. Тип: **Audience**
4. `Included Client Audience`: `rockfile-api`
5. `Add to access token` = ON
6. `Save`

После этого получите новый токен и проверьте, что `aud = rockfile-api`.

### Добавление Audience Mapper прямо в client

**Используйте этот вариант, если `rockfile-api-dedicated` не видно в Client scopes.**

1. `Clients` → `rockfile-api`
2. Откройте вкладку `Mappers` (или `Client scopes`, если в вашей версии так называется)
3. `Add mapper` → `By configuration`
4. Выберите **Audience**
5. Заполните:
  - `Included Client Audience`: `rockfile-api`
  - `Add to access token`: ON
  - (опционально) `Add to ID token`: ON
6. `Save`

После этого получите новый токен и убедитесь, что `aud` содержит `rockfile-api`.

- Сначала реализовать **простой чат‑бот**:

  - поиск контактов и создание взаимодействий прямо из диалога;
  - сценарии диалога и быстрые кнопки для типовых действий;

- Затем, при появлении потребности в более богатом интерфейсе, добавить **Telegram Mini App**:
  - использование того же API и той же авторизации;
  - переиспользование части веб‑интерфейса Rockfile внутри Telegram;
  - режим «расширенного» просмотра/редактирования контактов и взаимодействий.
